# Test plan
## Introduction
В данном документе будут описаны границы тестирования, инструментальные средства, подходы, тестовые сценарии, которые будут использованы при тестировании мобильного приложения Music Player. Здесь будут описаны функции для тестирования и сделаны выводы о соответствии проекта требованиям.
## Test items
Music Player - это мобильное приложение для прослушивания музыки. Оно должно предоставлять возможность слушать музыку на устройстве и иметь одинаковый графический интерфейс на устройствах с разными разрешениями экрана. Ниже представлены характеристики качества, которыми должно обладать приложение.
### Functionality

 - Графический интерфейс для просмотра всех аудиозаписей на устройстве
 - Возможность прослушивания аудиозаписи
 - Управление проигрыванием
 - Поиск аудиозаписей
 - Создание плейлиста и добавление в него аудиозаписей
 ### Usability
 
 - Интерфейс пользователя должен быть таким, чтобы сразу было понятно как пользоваться приложением
 - Возможность использования предоставляемых приложением функций разными способами
 ### Portability
 
 - Возможность легко добавить поддержку управления проигрыванием аудио не используя интерфейс приложения (с помощью медиа-кнопок на наушниках, Android Wear, Android Auto, Google Assistant)

## Features to be tested
В приложении будут тестироваться следующие функции:

 - Проигрывание аудиозаписей
 - Контроль проигрывания
 - Поиск аудиозаписей
 - Создание плейлиста
 
 ### Functional requirements
 
 - Реализовать страницу со списком альбомов и возможностью просмотра аудиозаписей альбома
	 
	Здесь будет тестироваться:
	
	 1. Группировка аудиозаписей по альбомам
	 2. Отображение списка аудиозаписей альбома

 - Реализовать возможность проигрывания аудиозаписи и управления проигрыванием

	Здесь будет тестироваться:
	
	1. Обновление интерфейса при остановке/смене/перемотке аудиозаписи
	2. Управление проигрыванием в фоновом режиме
	

 - Поиск аудиозаписей

	Здесь будет тестироваться:
	

	 1. Отображение результатов поиска
	 
### Nonfunctional requirements

 - интуитивно понятный интерфейс
 - приложение должно адаптироваться для разных разрешений экрана

## Test approach
Используется системное тестирование, при котором проверяется соответствие системы функциональным и нефункциональным требованиям.
## Test environment
Тестирование производится на Mi 8 Lite, версия Android 10
